```
harn devtools [--tool <name>] [--usage] [--window-days <n>] [--format table|json]
```

Reports the on-disk state of the AI coding agents Harnery is built for ‚Äî Claude
Code (`~/.claude`), Codex (`~/.codex`), and Cursor (`~/.cursor`) ‚Äî in one place:
logged-in status, plan / seat tier, auth expiry, session counts, and, where the
tool records them locally, rate-limit / quota windows.

Everything comes from files on disk. There is **no network call and no vendor
API**. Auth tokens are inspected only for their non-secret claims (email, plan,
expiry); the token strings themselves are never read into the output. A signal
a tool keeps server-side surfaces as blank with a note rather than a guess.

## What each tool exposes locally

| Signal | Claude Code | Codex | Cursor |
| --- | --- | --- | --- |
| Logged-in | ‚úÖ credential + expiry | ‚úÖ `auth.json` | ‚öÝÔ∏è login presence only |
| Plan / tier | ‚úÖ seat tier | ‚úÖ plan (JWT claim) | ‚ùå server-side |
| Auth expiry | ‚úÖ | ‚úÖ | ‚ùå |
| Session count | ‚úÖ transcripts | ‚úÖ rollouts | ‚úÖ workspaces |
| Quota / reset windows | ‚ùå (live via `/usage`) | ‚úÖ 5h + weekly, from `token_count` | ‚ùå server-side |
| Token totals (`--usage`) | ‚úÖ transcript scan | ‚úÖ best-effort | ‚ùå server-side |

Two honest gaps: **Cursor** keeps usage and billing entirely on `cursor.com`, so
locally you get login presence and a workspace count only; and true
**billing-cycle countdown** is not local for any of them ‚Äî Codex's rate-limit
`resets_at` windows are the closest local proxy.

## Options

| Flag | Effect |
| --- | --- |
| `--tool <name>` | Restrict to a tool (repeatable): `claude-code`, `codex`, `cursor`. Default: all three. |
| `--usage` | Also scan local transcripts for approximate token totals. Opt-in because transcripts can be gigabytes. |
| `--window-days <n>` | With `--usage`, only count transcripts modified within N days (default 7). |
| `--format <type>` | `table` (default) or `json`. |

Token totals for the last day, JSON, one tool:

```bash
harn devtools --tool claude-code --usage --window-days 1 --format json
```

The JSON envelope is `{ ok, generatedAt, windowDays, tools: ToolStatus[] }`,
where each `ToolStatus` carries `installed`, `loggedIn`, `account`, `plan`,
`rateLimitTier`, `authExpiresAt`, `sessions`, `lastActivity`, `quota[]`,
`tokensUsed`, and `notes[]`.
