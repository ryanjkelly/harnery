## File format

Settings live in [JSONC](https://github.com/microsoft/node-jsonc-parser): JSON with comments and trailing commas. A published JSON Schema gives editor autocomplete and inline validation in VS Code, Cursor, Zed, JetBrains, and anything else that respects `$schema`.

## Locations

The project settings file is `<project-root>/.harnery/config.jsonc`, and it is authoritative. CLI flags and `HARNERY_*` env vars override it per invocation (each field's env override is noted in the [schema reference](/reference/config-schema/)).

> A user-global `~/.config/harnery/config.jsonc` is planned but **not yet read** by the config reader — only the project file is consulted today.

## Schema

Reference the published JSON Schema at the top of your config:

```jsonc
{
  "$schema": "https://harnery.com/schemas/config.schema.json",

  // ... your settings here
}
```

VS Code (and most modern editors) read the `$schema` URL and provide autocomplete + validation as you type.

## Example

```jsonc
{
  "$schema": "https://harnery.com/schemas/config.schema.json",

  // Project identity surfaced to peers + in log lines.
  "project_name": "my-project",

  // Coord layer
  "coord": {
    "freshness_seconds": 600,  // how stale a peer heartbeat can get before sweep
    "name_pool": "default"     // "default" | "hurricane" | "custom-pool-id"
  },

  // Web UI
  "web": {
    "port": 9000,
    "bind": "127.0.0.1"
  },

  // Backup
  "backup": {
    "repo": "rclone:gdrive:harnery-backup/my-project",
    "schedule": "daily"
  }
}
```

See [reference / config schema](/reference/config-schema/) for the full field listing.
